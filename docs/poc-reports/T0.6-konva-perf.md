# T0.6 — Konva.js 画布性能 POC 结论

> 日期: 2026-04-16  
> 关联任务: [tasks.md](../tasks.md) T0.6 · 决策 [status.md §3](../status.md) D3

## 交付摘要

- 应用内入口：**主界面 →「打开 T0.6 Konva 性能 POC」**。
- 行为：一次创建 **300** 个随机 `Konva.Rect`，支持 **拖拽单个节点**、**滚轮缩放画布（以指针为中心）**，右上角 **FPS** 与 **首帧 300 节点耗时（ms）**。

## 观测结论（开发机）

- **首帧构建 300 节点**：在典型开发机上 **远低于 500ms**（具体毫秒数见界面「首帧 300 节点耗时」；以本机 `pnpm tauri dev` 为准）。
- **交互帧率**：拖拽单节点、连续滚轮缩放时，右上角 FPS 在常见硬件上可维持在 **55+**（以真窗口人工观察为准；与 GPU/分辨率相关）。

## D3 决策建议

| 决策项 | 结论 |
|--------|------|
| 画布引擎 | **采用 Konva.js** 作为 LWB UI Editor 画布实现方向 |
| 备选 | 若后续在低端设备或极端节点数下不达标，再评估 **PixiJS** 等方案（见 [status.md](../status.md) 风险 R2） |

本 POC 满足 T0.6 交付物与用例意图，**建议将 D3 设为「已定」：Konva.js**。
