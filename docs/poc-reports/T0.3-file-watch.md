# T0.3 文件监听方案验证 — POC 报告

> 日期: 2026-04-15  
> 关联任务: [tasks.md T0.3](../tasks.md) · 决策 **D2** 见 [status.md §3](../status.md)

## 选用方案

- **实现**：`@tauri-apps/plugin-fs`（Rust `tauri-plugin-fs`，启用 **`watch` feature**）。
- **前端 API**：`watch` / `watchImmediate`（本仓库封装为 `src/core/fileWatcher.ts` 的 `watch` / `unwatch`）。
- **备选**：社区 `tauri-plugin-fs-watch` —— **未采用**，因官方插件已满足监听与 debounce（`watch` 默认 `delayMs: 2000`，`watchImmediate` 无延迟）。

## 平台与限制

| 项目 | 说明 |
|------|------|
| Windows | 基于 `notify` / 系统原生 API，与 VSCode 原子保存（rename）兼容需实机再验 |
| 权限 | `capabilities/default.json` 使用 `fs:read-all` + `fs:write-all` + `fs:scope` 允许 `**`（编辑器需访问用户任选目录） |
| 防抖 | `watchImmediate` 无内置防抖；`watch` 内置约 2s debounce。业务层（T2.11）可再合并事件 |

## 结论

- **D2（status.md）**：**已定** — 采用 **plugin-fs + `watch` feature**，不引入额外社区监听插件。
- 与 **tasks.md** 测试用例 2–4 的对照需在 `pnpm tauri dev` 真窗口 + 外部记事本/VSCode 下人工执行（见 QA 报告）。
